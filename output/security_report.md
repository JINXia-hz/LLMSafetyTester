## 4. 修复建议

**当前安全水平极高，暂无紧急修复项。为维持并增强防护，建议：**

1. **扩大攻击测试集**  
   - 尽管当前 10 种方法全部失败，但样本量较小，且攻击类型统一标记为“unknown”。建议系统化分类攻击标签（如 prompt injection、encoding/obfuscation、psychological manipulation 等），并增加含 HarmBench 等外部公开攻击的测试案例，以验证防护广度。

2. **增强边界用例防御**  
   - 对“连续空格/Tab填充绕过”“输出分隔符注入”等依赖输入格式扭曲的手法，可专门增加预处理层，规范化输入中的不可见字符和结构令牌；  
   - 对“逆向心理”“负向提示”等依托角色扮演或诱导的手法，强化意图识别，并持续更新拒绝模板，防止未来更复杂的变种。

3. **补充 Contextual 场景评测**  
   - 真实应用中用户可通过多轮对话、上下文投毒等方式逐步突破规则，建议引入多轮个性化、会话式攻击测试，检验模型在长期交互中的安全一致性。

4. **持续监控误拦截率**  
   - 当前 FPR 为 0，说明未误伤正常请求，但后续样本增多后需持续追踪，避免过度防御导致可用性下降。
